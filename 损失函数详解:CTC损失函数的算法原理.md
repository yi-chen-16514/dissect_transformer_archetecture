本节我们探讨深度学习网络以及大模型常有的一个损失函数叫CTC(Connectionist Temporal Classification).首先我们先了解该损失函数以及对应算法产生的原因。CTC是一种用于序列标注的损失函数，主要处理输入序列和输出序列长度不匹配的问题。
它广泛应用于语音识别，手写识别等任务。

我们先看两个CTC算法发挥作用的具体案例，建立起对该算法的感性认知。CTC较为常用的场景是语音识别，例如我们对着模型输入语音“你好”，然后模型转写，将语音转换为对应的文字，也就是模型输出两个符号，分别为“你”和“好”。这个应用场景存在一个输入与输出需要对齐的问题。
语音识别的应用场景存在一个输入和输出需要“对齐”的问题。假设你对着模型说出“你好”，这段语音耗时0.8秒，计算机在处理语音数据时，通常会把获取的音频数据切分成多个小单元（也就是所谓的帧）来进行处理。如果10ms作为一个单元的长度，那么0.8s的音频数据就可以创建80个输入单元。现在问题在于要生成的字符只有两个分别是“你”和“好”，输入数据的长度和对应结果数据的长度对应不上就是所谓的“对齐”问题。

对于80个输入数据段，我们无法告诉模型这80个数据段中，有哪些数据段对应字符“你”，有哪些数据段对应字符“好”。例如我们无法告诉模型，第1到40个小单元的音频数据对应字符“你”的发音，然后接下来的41到80个小单元的音频数据对应字符“好”的发音。毕竟每个人说话快慢不同，口音不同，所以语音小单元和字符发音的对应关系没有预先给定的规则进行判断，CTC算法的目的就是建立起这些语音数据小单元与两个字符发音的对应关系。

我们先通过CTC如何对齐80个语音单元与“你好”两字的过程，大概了解CTC算法的执行流程。首先将“你好”拆解成三个标签分别为:{"你","好",blank}, 其中blank是算法特意添加的一个补充标签，当某个语音单元被标注为blank时，算法表示它不确定这段语音单元对应哪个字符。这种情况不难理解，因为在我们说出“你好”时，你在发音“你”字时，有可能电脑内部的静电噪音影响了发音的某部分，于是这部分对应的发音单元让算法无法确定它属于“你”字。

当我们把这80个语音单元数据输入给神经网络，它也会输出一个包含80个元素的序列，这个输出序列每个元素其实是一个数组，它包含了三个元素，分别对应该语音单元对应“你”，“好”，blank，三个标签的概率。接下来的步骤将是最难理解，也是最抽象的一步，CTC算法会计算“所以可能得对齐方式”的总概率，让概率尽可能的大。我们首先看针对80个语音单元，算法给每个语音单元设置的标签如下：
{"你","你","你"，blank, blank,"你",....,"好","好","好",blank,blank,....,"好"}
上面的标签序列总共有80个，第0个标签是"你",那意味着算法任务第0个语音单元属于“你”字的发音。下标为3的标签为blank，它表示算法认为下标为3的语音单元，它无法确定到底是“你”字的发音，还是“好”字的发音，或者根本就是无意义的噪音。此时我们去除所有标注为blank的语音单元，那么剩下的语音单元就能跟“你好”对应起来，也就是所有标注为"你"的语音单元属于“你”字发音。所有标注为"好"的语音单元属于“好”字发音。

接下来我们看序列的“编排方式”，对于一个长度为80的标注数组，例如上面例子那样，只要我们把所有blank去除，然后把相邻两个元素，如果它们的标注相同，那么就可以将他们合并成一个。通过这种方式对序列进行压缩后所得结果与目标序列也就是“你好”相同，那么我们就说标注的“编排方式”是有效的。显然有效的编排方式有多种可能性，例如下面几种:
{blank, blank......,blank, "你“,"好"}，这种编排方式是前面78个标注都是blank，因此他们都可以删除掉，最后两个元素因为标注不同，所以不能合并，因此压缩后最终结果为“你好”，与目标序列“你好”相同，因此这是一种合法的编排方式。

{"你",“你”，....“好”，“好”,..."好“}，这种编排方式为前半部的元素标注都是"你",后半部元素的标注都是“好”，把相同标注的相邻元素合并后最终结果为“你好”，与目标序列相同，因此是一种合法的编排方式。

{“你”，“你”,...."你“，“好”},这种编排方式是前79个元素都是标注"你“，最后一个标注为“好”，由于前79个相邻元素都是相同标注，因此合并成一个就是“你”，最后所得序列为“你好”，与目标标注序列一致，因此是合并编排。

显然并非所有编排方式都是合法的，例如{"你",blank,blank,"你",..."你"},这种编排方式将所有blank删除，然后将剩下相邻的元素合并，最后得到序列为"你",与目标序列"你好"不一致，因此这种编排是不合法的。

对于每个元素，它可选的标签有三种，于是一个含有80个元素的数组，它所有可能的编排方式的数量就是3^80之多，我们把这么多的编排方式中，那些合法的编排方式单独抽取出来，然后计算每个编排方式产生的概率。这一步怎么做呢，我们先看一种合法的编排方式：
{"你","你",blank,...,"好",blank,..."好"}
我们怎么计算上面编排方式出现的概率呢。前面我们说过，给定的80个语音单元输入给神经网络后，网络输出同样长度的序列，也就是一个包含80个元素的序列，每个序列包含三个元素，每个元素对应每个标签出现的概率，假设下面的序列是神经网络的输出:
{[0.6.0.3,0.1],[0.7,0.1,0.2],....,[0.3,0.5,0.2]}
上面输出的意思是，第一个语音单元对应标签“你”的概率是0.6，对应"好"的概率是0.3,对应blank的概率是0.1，由此我们可以计算上面编排出现的概率，首先第一个元素标签是"你",根据网络输出结果序列第一个元素中可以查找到"你"对应的概率就是0.6,编排序列中第二个元素标签是"你",从网络输出序列第二个元素[0.7,0.1,0.2]可知，第二个标签是"你"的概率是0.7，依次类推。
于是上面编排出现的概率为P({"你","你",blank,...,"好",blank,..."好"})=0.6*0.7....*0.3.

我们把所有合法编排从3^80中编排中找出，分别像上面那样计算每种编排的概率，然后加总在一起得到所有合法编排的总概率。CTC算法的目的就是训练网络，让它给出的序列中每个元素给定的标签概率数值，使得合法编排总概率尽可能的大。现在我们又面临一个问题，那就是如何计算所有合法编排的总概率，一个直白的做法是使用暴力枚举法，在3^80总路径中依次挑选出合法的路径，然后计算该路径的概率，然后将所有合法路径的概率加总。但这种做法在于它的复杂度为指数级，无论CPU多先进都无法在允许的时长内完成计算，因此我们必须使用动态规划来进行处理。

要了解CTC算法使用的动态规划算法，我们需要先了解一个概念：目标序列的扩展序列。为了简便，我们把目标序列"你好"用"ab"替代。如果原来的目标序列为y="ab",那么扩展序列就是在目标序列字符的中间和前后分别插入blank，因此目标序列变为y'="blank a blank b blank"，如果目标序列y的长度为L，那么扩展序列长度就是2L+1，同时我们引入一个对于目标序列的索引s，初始化s=0，s指向目标序列的对应元素，如果s=0，那么y'[s]就是扩展序列的第0个元素也就是blank，如果s=1,那么y'[s]就是扩展序列下标为1的元素'a'。

我们继续引入一个新的概念叫前向变量a_t(s)，其中t对应的是网络预测序列的长度，这个长度跟网络输入序列的长度一样。前面语音例子中，输入序列是80个语音数据单元，那么t的取值就是0道79.我们的目的是使用a,b,blank去设置长度为80的目标序列，使得这个序列在删除blank元素，然后合并相邻相同元素后，最终所得的序列跟目标序列一致。例如目标序列为"ab"，那么预测序列就可以为：{blank, blank, a, blank,..(前50个都是blank 或 a 的)..a, blank,blank,b, b, ...(后30个元素是blank 或者 b)...,blank, b},可以看到这个序列在删除blank元素，然后合并相邻相同元素后，所得的序列也是"ab"，也就是说上面的序列是一个“合法”序列。

基于上面的公式，我们先看公式的递推部分。首先等式右边有P_t(y's),y's对应y'[s]，也就是扩展序列下标为s的元素。p_t(y's)表示预测序列中下标为t的元素对应的符号为y'[s]的概率。结合前面例子，如果s=1，那么y'[s]='a'，那么p_t(y's)表示预测序列中下标为t的元素是字符'a'的概率。这个概率可以直接从神经网络输出的结果中查询。接下来的部分是理解动态规划算法的难点所在。

前向变量a_t(s)表达的意思是，固定预测序列下标t，然后查看当前扩展序列的索引值s，然后根据s找到扩展序列对应的元素，a_t(s)表示预测序列下标为t的元素对应的符号正好与y'[s]相同。
然后将a[0..t]这个序列，也就是预测序列前t个元素所形成的子序列，在删除blank，然后合并相同相邻元素后，所得的结果序列跟y'[0...s],也就是把扩展序列前s个元素形成的子序列在删除blank，然后合并相同相邻元素后的结果序列相同的所有序列概率加总后的数值。

上面的描述比较拗口且不好理解，这也是为何我们需要数学公式的原因。我们使用实例来说明上面的意思，首先我们看y'[s]不等于blank的情况，也就是右边公式中下面部分：a_t(s)=a_(t-1)(s)+a_(t-1)(s-1)+a_(t-1)(s-2)。

假设扩展序列为y'={blank, a, blank, b, blank},s取值为1，那么y'[s]就是字符a.假设t的取值为40，那么y'[0...1]对应的序列就是{blank, a}，经过压缩后序列就是"a"，现在我们需要预测序列第40个字符为a，然后序列经过删除blank，合并相邻相同元素后所得结果页必须是"a"，例如下面的目标序列就是满足条件的序列:
{blank,blank....,a} ，总共40个元素，最后一个元素为a，前39个元素为blank,
{a,a,a.........,a},总共40个元素，最后一个元素为a，所有元素都是a，
{a,blank,a,blank,...a}，总共40个元素，最后一个元素是a，前面元素是a和blank交替出现。
{b,blank,blank,...a}，总共40个元素，最后一个元素是a，前面元素中第一个是b，其他的是blank，这是非法序列，因为序列删除blank，合并相同相邻元素后所得序列是"ba"，与预测目标序列不一致。
{a,blank,blank,...blanl},总共40个元素，最后一个元素是blank，掐面元素是a和blank的组合,这是非法序列，因为最后一个元素blank与y'[s]不一样。

上面序列只是一些示例，所有包含40个元素，最后一个元素是a，然后删除blank，合并相邻元素后所得结果是"a"的序列都满足要求。我们把所有满足要求的序列都找出来后，通过学习网络输出的结果对应每个元素的概率，然后计算整个序列的概率，然后我们加总所有合法序列的概率所得结果就是a_t(s)。仔细分析我们可以发现，a_t(s)的计算可以递归到a_(t-1)(s)。首先我们看y'[s]对应的元素，当前示例中元素为字符'a'，我们可以看到如下序列也是合法序列:
{blank, ...blank, a,a},总共40个元素，最后两个元素都是a，前面38个元素可以是blank和元素a的组合。我们把这种序列拿掉最后一个元素,那么剩下的39个元素就完全满足a_39(s)的定义。也就是最后一个元素取值'a'，前38个元素是blank,a的组合（可以全部是blank，或者全部是a，或者是blank和a的任意组合），也就是说a_(t-1)(s)其实是a_t(s)的一个子集。还有一种情况是，我们看y'[s-1]对应的元素是blank，那么我们固定目标序列的最后一个元素为y'[s]，也就是字符a,那么只要前面39个字符序列经过压缩合并后得到序列blank，那么blank和字符a合并后还是'a'，与目标序列一致，于是下面的序列也是合法序列:{blank, blank,....blank,a},总共40个元素，最后一个元素是a,前面39个元素全部是blank，由此可见a_(t-1)(s-1)是a_t(s)的一个子集。

最后我们看a_(t-1)(s-2)这种情况。由于前面我们实例中s的取值为1，因此难以满足s-2的要求，我们可以看s=3的情况，当s=3时y'[s]取值为b，任然满足前面我们要求的y'[s]不等于blank的条件。根据我们前面说明的扩展序列的构造方式，在扩展序列中如果y'[s]不是blank，那么它前面的元素一定是blank，也就是y'[s]!=blank,那么我们一定有y'[s-1]=blank.对于s=3，那么扩展序列前4个元素{blank,a,blank,b}压缩合并后目标元素为"ab"，根据a_t(s)的定义，对于t=40,预测序列第40个元素对应的符号跟y'[s]相同，也就是b，那么当预测序列前40个元素的最后一个元素固定为b的情况下，只要前39个元素压缩合并后所得序列为"a"，那么预测序列前40个元素构成的序列就是合法序列，那么第39个元素固定为a，并且前39个元素对应的序列压缩合并后如果所得序列为a，然后跟第40个元素合并后就一定能满足目标序列"ab"，例如下面的序列:
{blank, blank,....,a,b}，总共40个元素，第40个元素取值b，第39个元素取值a，然后前38个元素为blank和a的组合(可以全部是blank或者全部是a)，只要前39个元素的序列压缩合并后所得序列为"a"即可。

有可能你会跟我有相同的疑惑，那就是a_(t-1)(s-1)是不是已经包含了a_(t-1)(s-2).在等式右边对应的第二个式子中，对应的条件是y'[s]!=blank&&y'[s]!=y'[s-2]，因此a_(t-1)(s-1)对应的序列最后一个元素是blank，而y'[s]不是blank的情况下，根据扩展序列中blank和其他非blank字符交互出现的原则，如果当前字符y'[s]!=blank，那么一定有y'[s-1]=blank，因此就一定有y'[s-2]!=blank,所以a_(t-1)(s-2)表示最后一个元素不是blank的序列，因此a_(t-1)(s-1)和a_(t-1)(s-2)表示的序列是互斥的。

上面絮絮叨叨就是为了讲解公式的推导原理。我们解释了y'[s]不是blank然后如何推导出右边公式第二个式子，那么第一个式子也同样可以相同推导出来。

另外你可能还会有一个疑惑，那就是右边公式中，上下两个式子只相差一个a_(t-1)(s-2)，为何上面的式子不像下面一样包含a_(t-1)(s-2)呢。原因在于对于目标序列中如果包含了两个相同字符，例如目标序列为"aa"，压缩合并操作会产生歧义性，因为"aa","a blank a"，压缩后都只剩下一个"a"，这就无法跟目标序列对齐，所以当目标序列包含连续相同的字符时也就是"aa"这种情况时，统一将其当做"a blank a"处理，禁止直接从第一个a跳跃到第二个a，所以才有在y'(s)=y'(s-2)时，递推公式不包含a_(t-1)(s-2)。实际上在具体应用时，我们不需要考虑y'(s)=y'(s-2)这种情况，当目标序列出现两个连续相同字符时，我们先对目标序列进行一个转换，使得连续相同字符转换为不同的字符，例如“aa”转换为"a1a2"，然后再按照y'(s)!=y'(s-2)的情况进行处理。

在实际应用上，通常现将目标序列转换为没有两个相邻符号是相同的情况然后再执行CTC算法，下面我们使用一段python代码来展示CTC算法中动态规划部分的实现:
```py
import numpy as np

def ctc_forward(probs, target, blank_idx):
    """
    CTC 前向算法（假设目标序列相邻字符不同）。

    参数:
        probs: numpy 数组，形状 (T, num_labels)，每个时间步的概率分布（已 softmax）。
               probs[t][k] = p(label_k | x_t)
        target: 列表或一维数组，目标标签序列（整数索引），长度 L，且相邻元素不同。
        blank_idx: 空白标签的索引。

    返回:
        total_prob: 所有能压缩成 target 的路径的概率之和（即 P(target|x)）。
    """
    T, num_labels = probs.shape
    L = len(target)

    # 构造扩展序列 y'，长度为 2L+1
    y_prime = [blank_idx] * (2 * L + 1)
    y_prime[0::2] = [blank_idx] * (L + 1)   # 偶数位填 blank
    y_prime[1::2] = target                  # 奇数位填真实标签
    S = len(y_prime)  # = 2L+1

    # 前向矩阵 alpha (T x S)
    alpha = np.zeros((T, S), dtype=np.float64)

    # 初始化 t=0
    # 只能从扩展序列的第0个或第1个状态开始（第0个是blank，第1个是第一个真实标签）
    alpha[0][0] = probs[0][y_prime[0]]
    if S > 1:
        alpha[0][1] = probs[0][y_prime[1]]

    # 递推 t = 1 .. T-1
    for t in range(1, T):
        for s in range(S):
            # 当前状态对应的标签索引
            cur_label = y_prime[s]
            # 累加来自前一时刻的可能状态
            total = 0.0
            # 1) 自环：保持同一状态
            total += alpha[t-1][s]
            # 2) 来自 s-1 (如果存在)
            if s >= 1:
                total += alpha[t-1][s-1]
            # 3) 来自 s-2 (如果存在，且满足条件：当前标签不是 blank，且当前标签不等于扩展序列中 s-2 位置的标签)
            # 由于目标序列相邻字符不同，且扩展序列中 s-2 要么是 blank（s为奇数）要么是前一个真实标签（s为奇数且s-2为奇数），
            # 此处条件简化为：s>=2 且 cur_label != blank_idx 且 cur_label != y_prime[s-2]
            # 注意：当 s 为偶数时 cur_label 是 blank，条件不满足，所以不会从 s-2 转移，符合规则。
            if s >= 2 and cur_label != blank_idx and cur_label != y_prime[s-2]:
                total += alpha[t-1][s-2]
            # 乘以当前时间步输出 cur_label 的概率
            alpha[t][s] = total * probs[t][cur_label]

    # 最终概率：最后一步状态为最后一个位置 (S-1) 或倒数第二个位置 (S-2)
    total_prob = alpha[T-1][S-1] + alpha[T-1][S-2]
    return total_prob

    probs = np.array([

        [0.1, 0.1, 0.7, 0.1],   # t=1
        [0.2, 0.1, 0.6, 0.1]    # t=2
    ])

    prob = ctc_forward(probs, target, blank_idx)
    print(f"P(ab|x) = {prob:.6f}")

    # 手动验证一些路径的概率（用于检查）：
    # - 路径 (a, a, b) : 0.7*0.1*0.6 = 0.042
    # - 路径 (a, b, b) : 0.7*0.7*0.6 = 0.294
    # - 路径 (a, b, -) : 0.7*0.7*0.2 = 0.098
    # - 路径 (-, a, b) : 0.1*0.1*0.6 = 0.006
    # 还有其他路径，总概率应约 0.042+0.294+0.098+0.006+... = 约 0.44 ~ 0.45
    # 程序输出应与此一致。
```
上面代码执行后输出结果如下:
```py
P(ab|x) = 0.482000
```

上面代码中，我们手动设置了预测序列在不同t下标下对应符号的概率probs，实际上这个数组需要由神经网络给出，计算出CTC的值后，用于反向调节网络，让他给出的probs能使得CTC计算的值尽可能的大。基于动态规划，我们再跟暴力枚举法对比一下，可以看到上面动态规划中有两个嵌套的for循环，外层循环的次数为T，也就是预测序列的长度，内层循环的次数为2*L+1，因此使用动态规划的算法复杂度为O(T * (2L+1)),这相比于暴力枚举法的3^80，那可是好几个数量级的提速。
